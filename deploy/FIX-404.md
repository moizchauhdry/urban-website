# Live site shows WordPress 404 (local works)

Local uses Vite dev middleware. Live needs Apache/Nginx + no WordPress page conflict.

## Fix 1 — Physical folder (most reliable on WordPress)

SSH:

```bash
mkdir -p /var/www/urbanelitelimo/connecticut-black-car-service

cp /var/www/urbanelitelimo/urban-app/../connecticut-black-car-service.htaccess \
   /var/www/urbanelitelimo/connecticut-black-car-service/.htaccess
```

Or upload `deploy/connecticut-black-car-service.htaccess` from this repo to that path.

The folder can be **empty** (no `index.php`). App stays in `urban-app/`.

```bash
chown www-data:www-data /var/www/urbanelitelimo/connecticut-black-car-service/.htaccess
```

Test: https://urbanelitelimo.com/connecticut-black-car-service/

## Fix 2 — Root `.htaccess` (if Fix 1 not used)

Paste `apache-wordpress-snippet.txt` at **line 1** of `/var/www/urbanelitelimo/.htaccess`, **before** `# BEGIN WordPress`.

Remove old rules for `connecticut-black-car-and-limo-service` or `moiz`.

## Fix 3 — WordPress admin

1. **Pages** → trash any page with slug `connecticut-black-car-service`
2. **Plugins → Redirection** (or Rank Math) → remove redirects for that URL
3. **Settings → Permalinks** → Save (flushes rewrite rules)

## Fix 4 — Nginx (`.htaccess` does nothing)

```nginx
location /urban-app/ {
    alias /var/www/urbanelitelimo/urban-app/;
}

location ~ ^/connecticut-black-car-service(/.*)?$ {
    rewrite ^ /urban-app/index.html break;
}
```

Then: `sudo nginx -t && sudo systemctl reload nginx`

## Blank page on `/connecticut-black-car-service/` (urban-app works)

The HTML loads (title shows) but React does not mount — usually old JS on the server or missing assets.

1. Rebuild and re-upload:
   ```bash
   npm run build
   rsync -avz --delete dist/ user@server:/var/www/urbanelitelimo/urban-app/
   ```
2. Hard refresh (Ctrl+Shift+R) or incognito.
3. DevTools → Network: `/urban-app/assets/index-*.js` must be **200**.
4. Ensure `connecticut-black-car-service/.htaccess` exists (Fix 1).
