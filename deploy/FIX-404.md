# Fix WordPress 404 on `/connecticut-black-car-service`

A **WordPress 404** means the request never reached your React `index.html`. Check these in order.

## 1. Files on the server

SSH in and run:

```bash
ls -la /var/www/urbanelitelimo/urban-app/
```

You must see `index.html` and an `assets/` folder.

If missing:

```bash
# On your Mac
cd /path/to/urban-website
npm run build
rsync -avz --delete dist/ root@YOUR_SERVER:/var/www/urbanelitelimo/urban-app/
sudo chown -R www-data:www-data /var/www/urbanelitelimo/urban-app
```

Test a static file in the browser:

`https://urbanelitelimo.com/urban-app/index.html`

- **Works** → files are there; fix `.htaccess` or Nginx (steps 2–3).
- **404** → upload path is wrong or files not deployed.

## 2. Apache `.htaccess` (most WordPress hosts)

Open `/var/www/urbanelitelimo/.htaccess`.

Paste the block from `apache-wordpress-snippet.txt` at **line 1**, before `# BEGIN WordPress`.

Remove any old rules that mention `moiz`.

Reload and test:

`https://urbanelitelimo.com/connecticut-black-car-service/`

## 3. Nginx (DigitalOcean one-click sometimes uses Nginx)

`.htaccess` does nothing on Nginx. Use `nginx-snippet.conf` in your site config, then:

```bash
sudo nginx -t && sudo systemctl reload nginx
```

## 4. WordPress page conflict

In **WP Admin → Pages**, find `connecticut-black-car-service`:

- **Trash** that page, or change its slug (e.g. `connecticut-old`), **or**
- You will keep getting the WP template instead of the React app.

Also check **Plugins → Redirection** (or Rank Math) for redirects involving this URL.

## 5. Quick server check

```bash
# File exists?
test -f /var/www/urbanelitelimo/urban-app/index.html && echo OK || echo MISSING

# Apache or Nginx?
apache2ctl -v 2>/dev/null || nginx -v 2>/dev/null
```

## Blank white page (title shows, no content)

1. **Do not use** `/urban-app/index.html` in the browser — use `/urban-app/` or the public URL below.
2. Re-upload the **full** `dist/` after `npm run build` (old `index.html` may point to missing JS files).
3. DevTools → **Console** and **Network**: check for red errors or 404 on `/urban-app/assets/index-*.js`.
4. Rebuild includes a fix for `index.html` in the path; redeploy after pulling latest code.

## Expected result

- Page loads (not WP 404 theme).
- DevTools → Network → JS from `/urban-app/assets/...` (status 200)
- Public URL: `https://urbanelitelimo.com/connecticut-black-car-service/`
